WordPress整站迁移，从XAMPP到LNMP，从Windows到Linux

发表于 2017-01-04 | 分类于 技术 | 阅读次数:

推荐使用整站迁移的方式进行WordPress网站的迁移，非常省事。

因为我之前使用的是Windows+XAMPP在家庭电脑上搭建的WordPress网站，近期因为申请了阿里云的半年免费套餐之后准备转战到阿里云，同时决定采用Linux+LNMP（我知道L就是Linux），原本以为环境配置会比较麻烦，没想到其实很简单：

1.ECS使用Ubuntu 32位的安装镜像（据说内存只有1G的ECS选用64位后安装MySQL会出错）

2.下载LNMP一键安装包，下载地址和安装教程可参考这里，因为免费套餐的ECS地域只能选北京，因此我只能使用青岛地区的外网链接地址来下载，平均速度只有10+k/s，花了我一个晚上。

3.备份数据库，得到sql文件，打包网站，得到zip文件，这个打包网站根本不用一些所谓的专门工具，直接用winrar把网站目录（XAMPP下一般默认文件夹为htdocs）压缩成zip

4.打开yourwebsite.com/phpmyadmin，新建同名数据库，使用sql文件导入数据库，修改下属option表中的siteurl和home至新地址

5.将zip文件传送至ECS上（我用的是邮箱附件方法，因为在此之前我在ECS上安装了远程桌面环境，具体做法可参考这里）

6.新建虚拟主机并进行伪静态化。具体参考这里

7.解压zip文件，将网站文件（htdocs下属文件，不包含htdocs本身）解压至home/wwwroot/域名文件夹home/wwwroot/default（已存在的index等文件可以移到到别处），删去wp-config.php文件

8.使用新地址访问WordPress网站，自动进入配置界面，填入MySQL信息，执行之后就能进入熟悉的登录界面了

如果之前未进行伪静态化处理，之后发现点击文章显示404错误，可以在设置-固定链接下使用朴素链接方式修复
